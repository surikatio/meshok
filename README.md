# Авто-лот

Десктопное приложение для автоматической публикации лотов на аукционе [meshok.net](https://meshok.net) через REST API.

Пользователь один раз заполняет параметры лота (название, категория, теги, описание, цена, дата старта и т.д.), указывает Excel-файл со ссылками на фотографии, и приложение публикует по одному лоту на каждую строку файла с заданной задержкой между публикациями.

## Возможности

- Публикация лотов через `POST https://api.meshok.net/sAPIv1/listItem`
- Несколько аккаунтов (имя → Bearer-токен), выбор аккаунта прямо в форме
- Шаблоны лотов: сохранение, загрузка, удаление; последний запуск восстанавливается автоматически
- Чтение списка ссылок на фото из Excel — одна строка = один лот, каждая колонка = одна фотография
- Сдвиг времени старта для каждого следующего лота на величину задержки
- Журнал публикаций (`история/`) и лог-файл (`log.txt`)
- Автообновление до новой версии через релизы GitHub (только в собранном `.exe`)

## Требования

- Python 3.11+
- Windows (приложение и сборка `.exe` ориентированы на Windows; собранный билд работает без установки Python)

## Установка и запуск

```powershell
pip install -r requirements.txt
python main.py
```

При первом запуске рядом с `main.py` (или рядом с `.exe`, если используется собранная версия) автоматически создаются:

- `settings.json` — настройки приложения (аккаунты, имя Excel-файла, цены доставки)
- `шаблоны/` — сохранённые шаблоны лотов
- `история/` — журнал отправленных лотов
- `log.txt` — лог-файл

Все настройки (аккаунты, цены доставки, имя Excel-файла) задаются прямо в интерфейсе приложения — отдельный конфигурационный файл не требуется.

## Использование

1. Откройте шестерёнку настроек и добавьте хотя бы один аккаунт (имя и Bearer-токен meshok.net), а также укажите имя Excel-файла со ссылками на фото.
2. Поместите Excel-файл рядом с приложением. Каждая строка файла — отдельный лот, каждая ячейка в строке — ссылка на одну фотографию этого лота.
3. На главном экране заполните параметры лота: название, ID категории, теги, описание, цена, дата старта (или `0` для "сейчас"), задержка между лотами, срок размещения, аккаунт.
4. При необходимости сохраните параметры как шаблон для повторного использования.
5. Запустите публикацию — приложение пройдёт по всем строкам Excel-файла, опубликует по одному лоту на строку и покажет live-лог со счётчиком успехов/ошибок. Публикацию можно остановить кнопкой «Стоп» — текущий лот будет завершён, и цикл прекратится.

### Формат шаблона

Шаблоны хранятся как однострочные `.txt`-файлы в `шаблоны/`, поля разделены `;`:

```
name;category_id;tags;description;price;date;sleep_time;longevity[;autoprod[;account]]
```

- `date` — `ГГГГ-ММ-ДД ЧЧ:ММ:СС` или `0` (подставится текущее время)
- `longevity` — один из `3, 5, 7, 10, 14, 21` (дней)
- `autoprod` — `0`/`1` (соответствует параметру meshok `antisniper`)
- `account` — имя аккаунта из настроек
- `шаблоны/last.txt` сохраняется автоматически после каждого запуска и подставляется по умолчанию при следующем

## Сборка standalone `.exe`

```powershell
.venv\Scripts\flet pack main.py --name "Avto-lot" --icon icon.ico --product-name "Авто-лот" --distpath dist
```

Результат — `dist\Avto-lot.exe` (~61 МБ), не требует установленного Python. Excel-файл нужно положить рядом с `.exe`; `шаблоны/`, `история/`, `settings.json`, `log.txt` создаются автоматически при первом запуске.

## Структура проекта

```
main.py                  # точка входа Flet, настройка логирования
core/
    paths.py             # get_app_dir() — базовая директория (exe или исходники)
    settings.py          # AppSettings — настройки приложения, settings.json
    templates.py         # LotData + CRUD для шаблонов
    excel_loader.py       # load_url_list() — чтение ссылок из Excel
    validator.py          # validate_lot_data() — проверка полей перед отправкой
    history.py            # save_history() — журнал отправленных лотов
    api.py                 # make_lot() — формирование запроса и POST на meshok.net
    meshok_api.py          # MeshokAPI — клиент API (на основе meshokteam/sAPI-py)
    updater.py             # проверка и установка обновлений из GitHub Releases
ui/
    lot_form.py            # главный экран — форма лота
    progress_view.py       # экран публикации — лог, прогресс, кнопка "Стоп"
    settings_view.py       # экран настроек — аккаунты, Excel-файл, цены доставки
    template_dialog.py      # диалог сохранения шаблона
tests/                     # модульные тесты (pytest)
```
